### комментарий в ТЗ
```
Нужно ли учитывать случаи, когда файл с данными большой, например несколько Гб?
Нет, не нужно, ограничений по памяти нет, всё можно читать в память.
```
Из-за этого комментария решил реализовывать через список словарей, т.к. удобнее обращаться по ключам = наименованию полей в таблице, соответственно в коде понятнее и меньше возможности допустить ошибку. Если думать о ресурсе - памяти и скорости - то нужно было реализовывать через список кортежей, учитывая что в данном приложении нет потребности в изменении информации.
т.е. изначально в константы с названием столбцов - положить значение индекса в кортеже(порядок столбцов).
Например:
country,year,gdp....
```
COUNTRY = 0
YEAR = 1
GDP = 2
```
соответственно обращение к элементам далее по коду будет понятнее, обращение по индексу быстрее(чем по ключу в случае словарей), памяти потреблять будет меньше.
```
row = ('China', 2023, 1394)
print(row[COUNTRY])
```
